원칙 1

코인 디자인 — Coin Design

GOHSY 코인 디자인 시스템은 골드 #D4AF37을 기준으로 한다. 각 코인은 6-노드 그래프 엠블럼을 중심으로, 페르소나별 고유 문양이 각인된다.

골드 그래디언트 (#F8E89E → #E5C158 → #D4AF37 → #A8842E → #5C4A1A)는 코인의 입체감과 귀족적 분위기를 동시에 전달한다. 코인 이미지는 항상 원형 크롭 + drop-shadow + 회전 애니메이션을 적용한다.

— 골드 코인 디자인의 원칙과 구현 상세입니다.

원칙 2

페르소나 아이덴티티 — Persona Identity

GOHSY 4 Coin Matrix — 배우 · Wife · Daughter · Guild — 각각의 컬러 시스템을 가진다. 각 페르소나 카드는 고유 --accent CSS 변수로 테마가 지정된다.

페르소나 카드는 hover 시 translateY(-12px) + scale(1.02) + 상단 악센트 바 확장 효과를 공유한다. 각 카드의 코인 아트는 180px 원형 (mobile 220px)으로 통일한다.

— 3 페르소나의 시각적 아이덴티티와 일관성 원칙입니다.

원칙 3

인터랙티브 경험 — Interactive

GSAP ScrollTrigger 기반 스크롤 진입 애니메이션은 모든 페이지의 기본 인터랙션 레이어다. data-reveal 속성으로 opacity:0 → 1, y:40 → 0 전환을 일괄 적용한다.

VanillaTilt 3D 호버 효과는 코인 이미지와 카드에 적용된다. Splitting.js 문자 단위 진입 애니메이션은 브랜드명과 주요 타이틀에 사용된다. 모든 전환은 cubic-bezier(.2,.8,.2,1)로 부드러움을 보장한다.

— 스크롤 · 호버 · 로딩 인터랙션 설계 원칙입니다.

원칙 4

콘텐츠 레이아웃 — Content Layout

타이포그래피 체계: Cinzel(serif) 헤딩 · JetBrains Mono 메타데이터 · Pretendard 바디. 헤딩은 gold gradient 텍스트 효과를, 메타데이터는 대문자 + 넓은 자간을 사용한다.

그리드 시스템: 4열 matrix-grid (섹션 네비게이션), 3열 persona-grid (콘텐츠 카드), 2열 ops-grid (운영 패널). 모든 섹션은 7rem 패딩 (mobile 4rem) + 1px #1a1a1a 상단 경계선으로 분리된다.

— 일관된 타이포그래피와 그리드 시스템 가이드라인입니다.
